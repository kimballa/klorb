# Log via stdlib logging with TextualHandler plus a per-session file, named with a coolname nonce

* Date: 2026-06-29 23:00
* Question: How should klorb configure logging so the same log statements work whether
  klorb is run headless (CLI) or inside a Textual TUI, while also leaving a durable record
  of each run?
* Answer: Use Python's standard `logging` module, configured once via `logging.basicConfig`
  with two handlers: `textual.logging.TextualHandler` (writes to the active Textual app's
  console when one is running, else falls back to stderr) and a `logging.FileHandler`
  pointed at a new file under `KLORB_STATE_DIR/session-logs/`. The file is named
  `yyyy-mm-dd-hh-mm-<nonce>.log`, where `<nonce>` is a two-word kebab-case slug (e.g.
  `dastardly-happy`) generated by the `coolname` library, to disambiguate files created
  within the same minute.
* Reasoning: `TextualHandler` is the mechanism Textual itself recommends for routing stdlib
  log records into a running TUI's console without any code needing to know whether a TUI is
  active; its built-in stderr fallback means the same setup also works for the headless CLI.
  A per-session file under `KLORB_STATE_DIR` gives every run a durable, independently
  reviewable log regardless of what's visible on screen. Naming files by minute risks
  collisions when multiple sessions start in the same minute, so a short random nonce is
  appended; `coolname` produces human-readable, pronounceable slugs (as opposed to a UUID or
  counter) that are easy to reference verbally or paste into a bug report.
